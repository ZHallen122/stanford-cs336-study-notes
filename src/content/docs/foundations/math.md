## 阅读公式的顺序

看到公式时，不要先逐字符解析。按这个顺序：

1. 找输出是什么。
2. 标出每个输入的 shape。
3. 用 shape 检查操作是否合法。
4. 最后再理解每个符号的数学意义。

## 向量与矩阵

- 标量：单个数，例如 loss。
- 向量：一列有顺序的数，例如一个 token 的表示。
- 矩阵：二维数表，例如一批 token 的表示或线性层权重。
- tensor：把上述概念推广到任意维度。

若 `X` 的 shape 是 `(batch, tokens, d_model)`，`W` 的 shape 是 `(d_model, d_out)`，则 `X @ W` 的 shape 是 `(batch, tokens, d_out)`。中间的 `d_model` 被消去。

## 导数、梯度与链式法则

导数回答：“输入发生很小变化时，输出大约怎样变化？”梯度把这个问题同时问给很多参数。链式法则让这个变化穿过一连串函数，是反向传播的数学基础。

## 概率与对数

语言模型输出候选 token 的概率分布。训练通常最小化正确 token 的负对数概率：正确答案概率越接近 1，惩罚越小。

:::note[为什么总有 log？]
连续概率相乘容易变成极小数字；取对数后乘法变加法，更稳定，也更容易优化。
:::
