## 固定页面结构

每篇课程笔记依次包含：

1. 这节解决什么问题
2. 在完整数据流中的位置
3. 给 SDE 的心智模型
4. 严格定义与 shape
5. 最小代码或手算例子
6. 常见误区
7. 理解检查
8. 官方材料与更新状态

结构固定可以减少读者在每页重新寻找信息的成本，也让维护者容易发现缺失部分。

## 术语规则

- 英文术语首次出现时立即解释，不把读者强制送去另一页。
- 短解释回答“是什么”，术语库再回答“为什么需要”和“与什么不同”。
- 使用缩写前先写全称，例如 Byte Pair Encoding (BPE)。
- 不连续堆叠三个以上未解释的新术语。

MDX 中可使用：

```mdx
import Term from '../../../components/Term.astro';

<Term
  term="token"
  slug="token"
  definition="模型处理文本时使用的离散单位。"
>token</Term>
```

## 阅读障碍友好

- 段落控制在 2–5 句，一段只表达一个主要观点。
- 使用描述性标题，避免连续大段文字。
- 不依赖颜色单独传达状态；颜色必须搭配文字或图标。
- 链接文字说明目标，避免“点这里”。
- 公式前先用自然语言说明输入与输出。
- 图片必须有替代文本；复杂图在正文中提供等价说明。
- 动画不是理解内容的唯一方式，并尊重 `prefers-reduced-motion`。

## 教学图片规范

图片必须承担解释任务，而不是只用于装饰。优先画数据流、组件关系、shape 变化、时间顺序和方案对比；一个短段落已经能说清的内容，不需要再画图。

### 选择格式

| 内容 | 首选格式 | 原因 |
| --- | --- | --- |
| 节点、箭头、流程、架构 | SVG | 文字和线条放大后仍清晰，可直接版本管理 |
| 简单的依赖关系草稿 | Mermaid | 容易修改，适合先确认信息结构 |
| 曲线和实验数据 | 由数据生成的 SVG/PNG | 图必须能从原始数据重建 |
| 界面或工具操作 | 截图 | 只截理解任务所需区域 |
| 概念隐喻或封面 | AI 生成位图 | 不能用来表达精确技术关系 |

### 图片内容

- 一张图只表达一个主要关系，标题直接说明读者应该看什么。
- 图中文字保持简短；解释性段落放在正文，不塞进图中。
- 节点按正文阅读顺序排列，箭头方向保持一致。
- 颜色必须同时搭配文字、形状或线型，不能让颜色成为唯一线索。
- 避免红绿对立；明暗主题下都要保持清楚边界。
- 手机宽度下不能依赖读者横向滚动才能理解主线。
- 横向图缩到 320–390px 后若文字太小，必须提供纵向移动版，并用 `<picture>` 在 `40rem` 以下自动切换。

### 无障碍与替代文本

- 每张信息图都必须有 `alt`，按阅读顺序概括它表达的关系，而不是写“流程图”。
- 图后提供简短 `figcaption`，说明读图重点、符号或适用边界。
- 正文必须包含图中的核心结论；无法看到图片的读者仍能继续学习。
- SVG 源文件包含 `<title>` 和 `<desc>`，但页面上的 `alt` 仍不可省略。
- 纯装饰图片使用空 `alt=""`，并且不能占据正文的主要位置。

### 文件与维护

- 可复用图放在 `src/assets/diagrams/`，使用小写短横线文件名。
- 图的源文件与使用它的讲义一起提交，不只提交导出的截图。
- 图中术语与正文保持一致；修改 shape、步骤或术语时同时搜索并更新相关 SVG。
- MDX 中使用 `<figure class="teaching-figure">`、`<img>` 和 `<figcaption>`，明确填写宽高，避免页面加载时跳动。

```mdx
import dataFlow from '../../../assets/diagrams/language-model-data-flow.svg';

<figure class="teaching-figure">
  <img
    src={dataFlow.src}
    width={dataFlow.width}
    height={dataFlow.height}
    alt="按阅读顺序描述图中的输入、变换与输出。"
  />
  <figcaption>说明读者应该注意的变量或边界。</figcaption>
</figure>
```

## 代码规则

- 示例尽量小，并标明输入、输出、shape、dtype 和 device。
- 先展示正确性版本，再展示优化版本。
- 避免横向滚动很长的行；长函数拆成可命名步骤。
- 报错示例除了“怎样修”，还解释“哪个不变量被破坏”。

## 内容归属

明确区分：课程官方事实、笔记作者的解释、个人实验结果和延伸材料。引用课程资料时链接原始来源；不要复制大段受版权保护内容。

## 发布前检查

- 键盘 Tab 可以到达所有交互元素。
- 放大到 200% 时页面没有遮挡正文。
- 明暗主题都有足够对比度。
- 只读标题也能理解页面结构。
- 所有内部链接与构建检查通过。
- 图片在 320px 宽度下仍可辨认，且 alt text 能独立表达核心关系。
