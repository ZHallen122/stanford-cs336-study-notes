## 目标地图

```text
文本 → tokenizer → token IDs
token IDs → embedding → Transformer blocks → logits
logits + targets → loss → gradients → optimizer update
```

开始模型与训练循环前，可先完成 [Lecture 2 · PyTorch 与资源核算](../../lessons/02-pytorch-resources/)；它解释 shape、autograd、optimizer state、checkpoint，以及怎样估算内存与 FLOPs，但不提供可提交实现。

## 建议里程碑

1. tokenizer 可以稳定 round-trip。
2. 单个模型组件通过 shape 与数值测试。
3. 完整模型完成一次 forward。
4. loss 与 backward 可运行，梯度有限且非零。
5. 很小的数据集可以被过拟合。
6. 完成最小训练并保存可复现记录。

## 遇到 NaN 时

- 找到第一个出现 NaN/Inf 的 tensor，而不是只看最终 loss。
- 检查学习率、初始化、归一化与 log/exp 的输入范围。
- 暂时缩小模型与 batch，固定随机种子。
- 比较优化前后的最小参考实现。

## 完成定义

不要只以“测试绿了”为完成。你还应能解释每个组件的数据 shape、参数量级、主要成本，以及一个最可能的失败模式。
