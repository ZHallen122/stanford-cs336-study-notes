这份笔记的目标不是把 CS336 变“简单”，而是把它变得**可以逐层理解**。课程官方要求深度学习、PyTorch、线性代数、概率与系统优化基础；这里会为缺少这些背景的 SDE 补一条坡度更缓的入口。

## 每篇笔记怎么读

<Steps>
1. **先看“这节解决什么问题”**：只建立方向，不急着记公式。
2. **再看“给 SDE 的心智模型”**：把新概念映射到编译器、数据库、分布式系统等熟悉领域。
3. **碰到点线术语时停一下**：聚焦或悬停即可看到短解释。例如 <Term term="token" slug="token" definition="模型处理文本时使用的离散单位；可能是一个字、词的一部分或标点。">token</Term>。
4. **做“理解检查”**：能用自己的话回答，就继续；不能就回到对应小节。
5. **最后再运行代码**：代码用于验证心智模型，不是用复杂实现掩盖概念。
</Steps>

## 你随时可以做的事

- 按 `/` 打开全站搜索，查术语或报错关键词。
- 打开右下角**阅读设置**，放大文字或开启高可读模式。
- 从左侧导航进入[术语库](../reference/glossary/)，再按浏览器后退回到原文。
- 页面太长时，只读右侧目录中的一个小节，然后休息。
