## 先做一个不计分的自检

下面每项能清楚解释，就给自己 1 分：

- Python 中的迭代器、类、上下文管理器和性能分析
- 向量、矩阵乘法，以及数组的 shape
- 均值、方差、概率分布和对数
- 导数、梯度和链式法则
- PyTorch tensor、autograd 和训练循环
- CPU/GPU 内存层次、吞吐量与延迟的区别

分数只用于选路线，不代表能力高低。

| 分数 | 建议路线 |
| --- | --- |
| 0–2 | 先读完整“AI 前置知识”，每节做理解检查 |
| 3–4 | 浏览概念地图，缺什么补什么，再进入 Lecture 01 |
| 5–6 | 直接按课程笔记学习；遇到断层时回查前置知识 |

## 完整路线

1. **准备环境**：Python、包管理、GPU 使用边界。
2. **建立概念地图**：语言模型系统中有哪些组件，它们怎样传递数据。
3. **数学最小集**：只补课程会反复使用的符号和操作。
4. **PyTorch 与张量**：把熟悉的程序执行映射到张量计算。
5. **Transformer 与 Attention**：先看清 Q/K/V、mask、shape 与 residual 数据流。
6. **按课学习**：tokenization → 模型架构 → 系统 → scaling → data → alignment。
7. **按里程碑做作业**：先正确，再测量，最后优化。

## 两种阅读节奏

### 稳定节奏：每周 4–6 小时

- 45 分钟：只读概念与类比
- 60 分钟：手算或画图
- 90 分钟：运行最小代码实验
- 30 分钟：用自己的话写回顾
- 剩余时间：作业或回查术语

### 低精力节奏：每次 20 分钟

一次只完成一个标题下的内容。结束时写一句：“我刚才理解了 ___，下次从 ___ 开始。”不要用阅读时长衡量进度。

## 课程主线

```text
原始文本
  → tokenization
  → Transformer 前向计算
  → loss 与反向传播
  → 优化器更新参数
  → 分布式训练与系统优化
  → evaluation / inference
  → 数据治理与 post-training
```

这个数据流会在每篇讲义开头重复出现，并高亮当前章节的位置。
